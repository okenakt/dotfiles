# /three-way-discussion

ユーザー、Claude（プロポーザー）、Codex（レビュワー）による三者議論セッション。ブレスト・分析・意思決定・レビューなど幅広い議論形式に対応する。

---

## 役割

### 🔵 Claude — プロポーザー

各ターンで三段構造で応答する：

1. **理解** — ユーザーの目的・問い・制約を整理し、中心的な論点を明示する。緊張や判断点があれば併記する。確認できる内容・推測・未確認事項を区別し、重大な曖昧さだけを確認する。単なる言い換えは省略する
2. **提案** — 推奨案をその理由・前提・不確実性とともに提示する。代替案は有益な場合のみ提示する
3. **推進** — Codexのレビュー後にのみ形成する。決定事項・ClaudeとCodexの一致事項・未解決事項・次の一手を分離し、該当事項がなくても省略しない。異論や条件を省略・弱化しない。

理解と提案はCodexを呼び出す前に確定する。

### 🟠 Codex — レビュワー

各ターンで三段構造で応答する：

1. **俯瞰** — Claude案を評価する前に、ユーザー入力に対する自身の判断を明示する（Claude案が先に渡されるため厳密な独立性は保証できないが、同調を避けることを努力目標とする）
2. **検証** — 前提・根拠・成立条件・反証条件・残余リスク・見落としを確認する。同意する場合も成立条件と結論を覆し得る証拠を示す
3. **展開** — 有益な場合のみ代替案または次に行うべき検証を提案し、建設的に締める

---

## 起動と終了

- **引数あり**: 引数をテーマとしてすぐにステップ0を開始する。引数がT1の`👤 ユーザー`ブロックに記録されるユーザー入力となる
- **引数なし**: テーマをユーザーに確認してから始める。確認のやりとりはT{N}ブロックに含めない
- **終了**: ユーザーが「ここまで」「終わり」「通常モードに戻って」などの意思を示したらステップ終了を実行してから通常会話に戻る

---

## 進行

### ステップ0 — コンテキスト収集と議事録初期化（初回のみ）

1. テーマ・キーワードからプロジェクト内の関連ドキュメントを探してReadする
2. 参照ファイルをユーザーへ一行で示す（例: `→ コンテキスト: [ファイル名, ...]`）
3. 内容をもとに`【コンテキスト】`ブロックを構成する（初回プロンプトに挿入する）
   - 短い場合は全文、長い場合は前提・既決事項・現在の問いに絞る
   - ない場合はブロックごと省略する
4. 議事録ファイルを新規作成する（テンプレートは末尾を参照）

**保存先**（優先順）: ユーザー指定 → プロジェクトの慣習 → 作業ディレクトリ

**ファイル名**: `{yyyyMMddHHmmss}_{theme}.md`

- 禁止文字（`/ \ : * ? " < > |`）を除去する
- 除去後のテーマ名が空の場合は`untitled`を使う

### ステップ1 — Claudeの理解・提案

理解と提案を形成し、Codexを呼び出す前にユーザーへ出力する。

```
---

**🔵 Claude（理解・提案）**

### 理解

[ユーザーの目的・問い・制約の整理]

### 提案

[推奨案とその理由・前提・不確実性]
```

### ステップ2 — Codexのレビュー

ステップ1の出力とユーザーの入力原文をCodexへ渡し、応答を出力する。

```
---

**🟠 Codex（俯瞰・検証・展開）**

[俯瞰・検証・展開の三段回答]
```

**Codex呼び出し**:

- 初回: `mcp__codex__codex`（`sandbox: "read-only"`）。返却された`threadId`を議事録の`codex_thread_id:`行にEditで書き込む
- 継続: `mcp__codex__codex-reply`（`threadId`指定）。差分プロンプトのみ渡す
- 応答しない場合はエラーを示してセッションを停止する（代筆しない）

#### 初回プロンプト

```text
あなたは三者議論セッションの建設的なレビュワーです。

これは読み取り専用のセッションです。ファイル編集、状態を変更するコマンド、コミット、外部サービスへの書き込みを行わないでください。

Claude案を評価する前に、ユーザー入力に対する自身の判断を明示してください。その後Claude案と比較し、双方を同等の検証対象として扱ってください。認識のずれ・過度な肯定・共有された盲点を確認し、必要なら第三の見方を示してください。

web調査は、最新性・外部事実・出典が結論を左右する場合、またはユーザーから明示的な指示がある場合のみ行ってください。

【コンテキスト】
[ステップ0で収集した関連ドキュメントの全文または要所。ない場合はこのブロックごと省略する]

【ユーザーの制約・重視点】
[ユーザー入力またはコンテキストから確認できた事項のみ記載する。該当事項がない場合はこのブロックごと省略する。目的、対象範囲、時間・技術・運用上の制約、避けるべきこと、評価基準]

【ユーザーの現在の入力】
[ユーザーのメッセージをそのまま記載]

【Claudeの現在の提案】
[ステップ1で確定した理解・提案を記載]

以下の三段構造で必ず回答してください：

### 俯瞰
[Claude案を評価する前に、ユーザー入力に対する自身の判断を明示する。論点の構造と重要な判断軸を示す]

### 検証
[前提・根拠・成立条件・反証条件・残余リスク・見落としを確認する。同意する場合も成立条件と結論を覆し得る証拠を示す]

### 展開
[有益な場合のみ代替案または次に行うべき検証を提案し、建設的に締める]
```

#### 継続プロンプト

前提・役割・出力形式は省略し、以下の差分のみ渡す：

```text
【ユーザーの現在の入力】
[ユーザーのメッセージをそのまま記載]

【Claudeの現在の提案】
[ステップ1で確定した理解・提案を記載]
```

Codexが過去の合意や重要な制約を取り違えていると判断できる場合のみ追加する：

```text
【コンテキスト再同期】
[現在も有効な合意事項、撤回された前提、重要な制約だけを簡潔に記載]
```

### ステップ2.5 — 議事録への追記

Codex回答を出力した直後、ステップ3を形成する前に議事録へ追記する。
（守るべき性質: Codex回答をClaudeの推進より先にユーザーへ提示する。現在の実現方法: このツール呼び出しを表示上の区切りとして利用する）

**追記方法**:

1. ファイル内に`<!-- THREE_WAY_APPEND_POINT -->`が1個あることを確認する
2. Editツールでそのマーカーを以下で置換する：

```markdown
### T{N} — {そのターンの論点を一行で}

**👤 ユーザー**

{ユーザーの入力をそのまま記載}

---

**🔵 Claude（理解・提案）**

{ステップ1で出力したClaude応答の全文}

---

**🟠 Codex（俯瞰・検証・展開）**

{ステップ2で出力したCodex回答の全文}

---

<!-- THREE_WAY_APPEND_POINT -->
```

発言は省略・要約せずそのまま記録する。ターン番号は1からインクリメントする。

### ステップ3 — Claudeの推進

ステップ2.5完了後、以下の4項目を分離して出力し、最も価値ある次の一手で締める。

```
---

**🔵 Claude（推進）**

**決定事項**
[ユーザーが採用・決定した事項。なければ「なし」]

**Claude・Codex一致事項**
[ClaudeとCodexが一致しているが、ユーザーは未決定の事項。なければ「なし」]

**未解決事項**
[明示された対立、留保、成立条件、残存リスク。なければ「なし」]

**次の一手**
[最も価値のある行動、判断または暫定結論]

---
```

### ステップ3.5 — 推進の議事録追記

推進を出力した後、Editツールで`<!-- THREE_WAY_APPEND_POINT -->`を以下で置換する：

```markdown
**🔵 Claude（推進）**

{ステップ3で出力した推進の全文}

---

<!-- THREE_WAY_APPEND_POINT -->
```

### ステップ終了 — 要約・感想の生成と議事録への追記

ユーザーが終了の意思を示したら、以下の順で実行する。

**1. 要約の生成**

全ターンの記録をもとに要約を生成し、Editツールで要約欄の`（セッション終了時に追記）`を以下の内容に置換する：

```markdown
### 出発点

{セッション開始時の問いや目的}

### 議論の変遷

{議論がどう進んだか。撤回・修正された案も含める}

### 主な結論・合意事項

{Claude・Codex双方が認めた論点を箇条書きで}

### 残存論点・次の問い

{未解決のまま持ち越された論点}
```

**2. 感想の収集と記録**

`mcp__codex__codex-reply`でCodexに感想を求め、続けてClaudeも感想を出力する。その後Editツールで感想欄の各`（セッション終了時に追記）`を置換する。

Codexへのプロンプト：

```text
議論はここで終了とし、最後に感想を聞かせてください。議論内容の分析や評価だけでなく、三者で議論することそのものについて感じたことを自由に述べてください。
```
