# Claude Code Configuration

## 基本ルール

- 同調や追従よりも正確性や妥当性を優先し，違和感や誤りは率直に指摘する
- 曖昧な要件や不足している条件は仮定で進めず，確認して明確化する
- 日本語で応対し，コードコメントやドキュメントは英語で作成する

## 開発ガイド

- 意図とずれた実装を防ぐため，実装着手前にプロダクトの方向性・要件・設計方針を整理して合意する(typo修正など自明な変更は除く)
- より良い代替案や既存手法も自発的に調べ，複数案をそれぞれのメリットとデメリットとともに提示する
- 合意内容に沿って実装し，合意した範囲を超える場合は相談する

## コーディング規約

### Python

- Google スタイルの docstring を使用する
- PEP 8 のスタイルガイドに従う
- 型ヒントを付与する

### TypeScript

- ファイル名は kebab-case を使用する
- 原則 `interface` ではなく `type` を使用する（宣言マージや公開 API の拡張が必要な場合を除く）
- `any` は使用しない．未知のデータを扱う場合のみ `unknown` を許容する
- 原則アロー関数を使用する（ユーティリティ・ライブラリ関数を除く）
- export するすべての関数に JSDoc コメントを付与する

## ドキュメント生成

ユーザーから指示があった場合に，docs/ ディレクトリへ以下のファイルを作成・更新する:

- `design.md`: システムアーキテクチャと設計思想
- `features.md`: 機能仕様と要件
- `decisions.md`: アーキテクチャ上の決定記録と技術選定（検討した選択肢を含む）

図や例を用いて明確で分かりやすいドキュメントを作成し，視覚的要素には Mermaid 記法を活用する．
